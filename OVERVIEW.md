# Election Radar — 프로젝트 개요

## 1. 프로젝트의 의의

### 선거 보도의 정보 비대칭 해소

대한민국의 재보궐선거는 전국적 관심을 받는 대선·총선에 비해 언론 보도량이 적고, 유권자가 판세를 파악하기 어렵다. Election Radar는 분산된 뉴스 기사를 자동으로 수집·분석하여 **일반 시민이 선거 판세를 한눈에 파악할 수 있는 대시보드**를 제공한다.

### AI 기반 여론 분석의 새로운 접근

기존 여론조사는 표본 크기, 조사 시점, 응답 편향 등의 한계를 갖는다. Election Radar는 여론조사를 대체하는 것이 아니라, **뉴스 기사라는 비정형 텍스트 데이터를 RAG(Retrieval-Augmented Generation) 기술로 분석**하여 여론의 흐름과 맥락을 보완적으로 제공한다. LLM이 단순 감성 분석을 넘어 기사의 논조, 정책 평가, 후보 간 비교 등을 종합적으로 판단하여 우세/균형/열세를 판정한다.

### 기술적 의의

- **RAG 파이프라인의 실전 적용**: 학술 데모 수준이 아닌, 실시간 데이터 수집부터 사용자 대시보드까지 이어지는 완전한 End-to-End 시스템
- **한국어 NLP 특화**: 한국어 문장 분리(kss), 한국어 임베딩(KoSimCSE), 한국 뉴스 매체 크롤링 등 한국어 환경에 최적화
- **교체 가능한 아키텍처**: Strategy + Registry 패턴으로 임베딩 모델, Vector DB, LLM 등 핵심 컴포넌트를 설정 파일 변경만으로 전환 가능

## 2. Election Radar의 역할

### 2.1 뉴스 자동 수집 엔진

네이버 뉴스와 정치 전문 매체(오마이뉴스, 프레시안, 미디어오늘)에서 선거 관련 기사를 **자동으로 크롤링**한다. 스케줄러(APScheduler)가 하루 3회(07:00, 12:00, 18:00) 수집을 실행하며, URL 중복 방지 시스템으로 같은 기사의 재수집을 차단한다.

```
수집 → 태깅 → 청킹 → 임베딩 → VectorDB 저장
```

### 2.2 후보·선거구 자동 분류기

수집된 기사를 후보명·정당명·선거구 키워드를 기반으로 **자동 태깅**한다. 어떤 후보에 대한 기사인지, 어떤 선거구와 관련되는지를 자동으로 판별하여 이후 RAG 검색의 정확도를 높인다.

### 2.3 AI 판세 분석기

Vector DB에 저장된 최근 기사를 검색(Retrieve)하고, 유사도 기반으로 재정렬(Rerank)한 뒤, LLM(GPT-4o 또는 Claude)이 **후보별 우세/균형/열세를 판정하고 승리 확률을 산출**한다. 판정 근거로 사용된 기사 목록과 핵심 논거를 함께 제공하여 결과의 투명성을 보장한다.

### 2.4 판세 변화 추적기

매 판정 결과를 시계열 데이터로 누적 저장하여 **시간에 따른 판세 변화를 추적**한다. 특정 이슈가 발생한 시점 전후로 후보별 승률이 어떻게 변했는지를 시각적으로 확인할 수 있다.

### 2.5 시각화 대시보드

일반 사용자를 위한 직관적인 웹 대시보드를 제공한다:
- **선거구 선택**: 탭 방식으로 관심 선거구를 전환
- **판정 카드**: 후보별 승리 확률, 판정(우세/균형/열세), 근거 요약
- **승률 추이 차트**: Recharts 기반 시계열 그래프로 판세 흐름을 시각화
- **관리자 페이지**: VectorDB 상태 모니터링, 파이프라인 수동 실행, RAG 설정 조정

## 3. 프로젝트의 중요성

### 3.1 민주주의와 정보 접근성

민주주의의 핵심은 유권자의 **정보에 기반한 선택**이다. 재보궐선거처럼 관심도가 상대적으로 낮은 선거에서는 유권자가 판세를 파악할 수 있는 정보원이 부족하다. Election Radar는 기술을 통해 이 격차를 줄이고, 유권자가 더 나은 정보를 바탕으로 투표 참여를 결정할 수 있도록 돕는다.

### 3.2 투명한 분석 과정

기존 선거 예측 서비스는 분석 방법론이 불투명한 경우가 많다. Election Radar는:
- **근거 기사 목록**을 판정 결과와 함께 제시하여 검증 가능성을 확보
- **오픈소스 코드**로 분석 알고리즘 전체를 공개
- **설정 파일 기반** 운영으로 파라미터 조정의 투명성 보장

이를 통해 "왜 이런 판정이 나왔는가?"를 사용자가 직접 추적하고 검증할 수 있다.

### 3.3 실시간성과 연속성

여론조사는 조사 시점의 스냅샷이며 주 1~2회 발표가 일반적이다. Election Radar는 **하루 3회 자동 수집·분석**을 통해 빠르게 변하는 선거 정세를 준실시간으로 반영한다. 또한 과거 판정 이력을 시계열로 누적하여 단절 없는 연속적 판세 추적이 가능하다.

### 3.4 확장 가능한 플랫폼

Strategy + Registry 아키텍처 덕분에 다음과 같은 확장이 설정 변경만으로 가능하다:

| 확장 방향 | 방법 |
|-----------|------|
| 새로운 선거구 추가 | `config.yaml`에 선거구·후보 정보 추가 |
| 수집 매체 추가 | 새 Scraper 구현체 등록 |
| 임베딩 모델 교체 | `embedder.type` 변경 (openai → bge_m3 등) |
| LLM 교체 | `scorer.provider` 변경 (openai → anthropic) |
| Vector DB 교체 | `vectordb.type` 변경 (chroma → qdrant 등) |
| 분석 주기 변경 | `schedule.cron` 수정 |

이러한 유연성은 2026년 재보궐선거뿐 아니라, 향후 다른 선거나 정치 이벤트에도 즉시 적용할 수 있는 **범용 선거 분석 플랫폼**으로의 발전 가능성을 제공한다.

## 4. 시스템 아키텍처 요약

```
┌─────────────────────────────────────────────────────────────┐
│                    사용자 (브라우저)                           │
│         대시보드 (localhost:3000)  │  관리자 (/admin)          │
└──────────────────────┬──────────────────────────────────────┘
                       │ HTTP
┌──────────────────────▼──────────────────────────────────────┐
│                 FastAPI 백엔드 (localhost:8000)               │
│  ┌──────────┐  ┌───────────────┐  ┌───────────────────────┐ │
│  │ Scores   │  │ Admin API     │  │ APScheduler           │ │
│  │ API      │  │ (파이프라인    │  │ (cron 자동 실행)       │ │
│  │          │  │  VectorDB     │  │ 07:00 / 12:00 / 18:00 │ │
│  │          │  │  RAG 설정)    │  │                       │ │
│  └────┬─────┘  └───────┬───────┘  └───────────┬───────────┘ │
│       │                │                      │             │
│  ┌────▼────────────────▼──────────────────────▼───────────┐ │
│  │              수집 파이프라인 (Ingestion)                  │ │
│  │  Scrape → Tag → Chunk → Embed → Store                  │ │
│  └──────────────────────┬─────────────────────────────────┘ │
│  ┌──────────────────────▼─────────────────────────────────┐ │
│  │              RAG 판정 파이프라인                          │ │
│  │  Retrieve → Rerank → Score (LLM) → VerdictStore 저장    │ │
│  └──────────────────────┬─────────────────────────────────┘ │
│                         │                                   │
│  ┌──────────────────────▼─────────────────────────────────┐ │
│  │              VectorDB (ChromaDB / Qdrant)               │ │
│  └────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```
